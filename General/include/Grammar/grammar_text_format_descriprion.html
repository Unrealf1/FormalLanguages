GRAMMAR text presentation format

Note: this is still work in progress

The first line is optional. It is line of parameters, starting with "Parameters: "
Each parameter must not have more than one entry in this line.
Following description is valid for empty(or absent) parameters line.

At the top of the file there's caption "Terminals(%u):", where '%u' stands for number
of terminals grammar have. This line should be followed by same amount of lines in each
is another terminal. Spaces, line foldings and '-' symbol are not allowed in terminals(and non-terminals).

Then there is similar section, starting with "Non-terminals(%u):" line. But every
starting non-terminal should be followed by "-S" without a space.

Important: names of terminals and non-terminals should not intersect

The last section consists of "Rules(%u):" line and same number of rules(one line = one rule)
Lets refer to both terminal and non-terminal as item. Then rules are described like this:
item1 item2 ... itemN - itemN+1 itemN+2 ... itemN+K
where N is number of items in the left part of the rule and K is number of item in the right part.
Important: if there is only epsilon(nothing) in the right part, then it should be left
like this: "item1 item2 ... itemN - "

Important: each line should be no longer than 4096 bytes.

Parameters should be written in one line like this: Parameters: Name1=value Name2 Name3 Name4=value

Possible parameters
Name Possible value Value by default Description
NoNumbers No possible value* Not stated If stated, number of lines in each section is not required.
New section will be detected by key line (Terminals/Non-terminals/Rules)
And \0 is expected after the last section
NoStarting No possible value* Not stated If stated, '-S' suffix is not required to starting terminals. Only the
first non-terminal will be considered starting
*No possible value means that the parameter can only be stated or not stated in
the first line and '=value' is not allowed after it

Example(without parameters):

Terminals(2):
a
b
Non-terminals(1):
A-S
Rules(3):
A -
A - A A
A - a A b

Example(with parameters):

Parameters: NoNumbers NoStarting
Terminals:
a
b
Non-terminals:
A
Rules:
A -
A - A A
A - a A b
